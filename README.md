1. 项目概述
1.1 项目名称
AI 智能对话助手（智谱免费版）

1.2 项目目标
开发一个免费、可用的网页应用，调用智谱 GLM-4.7-Flash 模型，让用户进行 AI 对话。

1.3 选择理由
✅ GLM-4.7-Flash 完全免费

✅ 编程能力突出，适合技术问答

✅ 200K 上下文，可处理长文本

✅ 注册即送 2000 万 tokens

2. 功能需求
2.1 核心功能（P0 - 必须实现）
功能	描述
对话交互	用户输入问题，AI 返回回答
多轮对话	保留历史上下文，连续对话
会话管理	新建对话、清空当前会话
2.2 次要功能（P1 - 建议实现）
功能	描述
流式输出	AI 逐字输出，打字机效果
复制回答	一键复制 AI 回答内容
停止生成	中途停止 AI 回复
2.3 增强功能（P2 - 可选）
功能	描述
历史记录	保存对话到本地，刷新不丢失
主题切换	深色/浅色模式
Token 统计	显示本次对话消耗的 Token

4. API 对接说明
4.1 智谱 API 基本信息
项目	内容
API 地址	https://open.bigmodel.cn/api/paas/v4/chat/completions
模型名称	glm-4.7-flash
认证方式	Bearer Token（API Key）
请求方式	POST
输出格式	JSON
4.2 获取 API Key 步骤
访问：https://open.bigmodel.cn/

注册账号

进入「API Keys」页面

点击「创建 API Key」

复制保存（只显示一次）

4.3 请求参数
json
{
  "model": "glm-4.7-flash",
  "messages": [
    {"role": "user", "content": "你好"}
  ],
  "temperature": 0.7,
  "max_tokens": 2000,
  "stream": false
}
4.4 响应格式
json
{
  "choices": [
    {
      "message": {
        "content": "你好！有什么我可以帮你的吗？"
      }
    }
  ],
  "usage": {
    "total_tokens": 50
  }
}
5. 接口设计（后端 API）
5.1 对话接口
项目	内容
路径	POST /api/chat
请求体	{ "messages": [...], "stream": false }
响应	{ "success": true, "message": "xxx", "tokens": 150 }
5.2 流式对话接口
项目	内容
路径	POST /api/chat/stream
响应格式	Server-Sent Events (SSE)
响应示例	data: {"content": "你"}
5.3 健康检查
项目	内容
路径	GET /api/health
响应	{ "status": "ok" }

9. 验收标准
序号	验收项
1	用户可以输入问题并获得回复
2	对话上下文连续（AI 记得之前聊的）
3	API Key 不在前端代码中出现
4	网页在手机和电脑上正常显示
5	错误时有友好的提示信息
10. 风险与应对
风险	应对措施
智谱算力紧张，响应变慢	添加加载动画，提示用户等待
API 调用失败	显示友好错误信息，支持重试
免费版 QPS 限制	个人使用不会触发，如需并发可加队列
